# Чек-лист проверки сайта по SDD

Этот документ используется как приёмка сценариев из `docs/sdd.md`.

## SDD-1. Просмотр лендинга

- Проверить, что все якорные разделы открываются из хедера: `#map`, `#catalog`, `#about`, `#gallery`, `#faq`, `#contacts`
- Убедиться, что фиксированное меню доступно при любом скролле
- Убедиться, что hero-блок корректно сегментирует аудиторию: семьи, студенты, бизнес
- Проверить, что FAQ, партнёры, форма заявки и контакты отображаются без пустых состояний

## SDD-2. Изучение объекта через каталог

- Проверить, что в каталоге у каждой карточки есть цена, длительность, город и CTA
- Проверить, что фильтры каталога работают и не вылезают за контейнер
- Проверить, что каждая карточка объекта открывается по кнопке `Подробнее`
- Проверить модальное окно: описание, адрес, формат, график, безопасность, состав услуги
- Проверить, что кнопка `Забронировать объект` подставляет объект в форму
- Проверить закрытие модального окна по кнопке закрытия

## SDD-3. Изучение объекта через карту

- Проверить наличие `VITE_YANDEX_MAPS_API_KEY` в локальном окружении
- Убедиться, что карта Яндекс загружается без ошибок и центрируется на Саратовской области
- Проверить маркеры объектов и popup с краткой информацией
- Проверить, что ссылка из popup открывает подробную карточку объекта в каталоге
- Убедиться, что при отсутствии API-ключа показывается понятный fallback-блок

## SDD-4. Бронирование объекта

- Проверить отправку формы на `POST /api/bookings`
- Проверить наличие чекбокса согласия на обработку ПДн
- Убедиться, что после успешной заявки создаётся запись со статусом `new`
- Проверить, что форма не выходит за экран на мобильных устройствах

## SDD-5. Вход администратора

- Открыть `/admin`
- Проверить сценарий входа с `ADMIN_LOGIN` и паролем, соответствующим `ADMIN_PASSWORD_HASH`
- Проверить `POST /api/admin/login`, `GET /api/admin/session`, `POST /api/admin/logout`
- Проверить, что без токена admin-маршруты недоступны

## SDD-6. Workflow заявок в admin

- Проверить загрузку списка бронирований
- Проверить поиск заявок по имени, контакту, формату, `objectSlug` и комментарию
- Проверить фильтрацию по статусу
- Проверить workflow `new` -> `contacted` -> `confirmed` или `canceled`
- Проверить сохранение комментария менеджера
- Проверить сохранение данных в `server/data/catalog.sqlite`

## SDD-7. Экспорт заявок в CSV

- Проверить `GET /api/bookings/export.csv`
- Проверить экспорт текущей выборки из admin-панели в CSV
- Проверить, что CSV учитывает активные `search` и `status`

## SDD-8. CRUD объектов в admin

- Проверить загрузку каталога объектов из backend
- Проверить `GET /api/objects` и `GET /api/objects/:slug`
- Проверить `POST /api/objects`, `PUT /api/objects/:slug` и `DELETE /api/objects/:slug`
- Проверить создание, редактирование и удаление объекта из панели
- Проверить, что изменения отражаются в каталоге и на карте

## Нефункциональная проверка

- Проверить ширины `390px`, `375px`, `360px`, `320px` без горизонтального скролла
- Проверить, что заголовки карточек каталога переносятся и не ломают сетку
- Проверить, что карта Яндекс корректно масштабируется и маркеры не обрезаются по краям
- Проверить клавиатурную навигацию по кнопкам, ссылкам и полям формы
- Проверить контрастность текста на стеклянных карточках
- Проверить `aria-label` на кнопке мобильного меню и маркерах карты
- Проверить наличие `.env` на основе `.env.example`
- Проверить, что `ADMIN_PASSWORD_HASH` задан и plain-text пароль не хранится в коде
- Проверить генерацию hash через `npm run admin:hash-password -- "password"`
- Проверить, что `.env` и `.env.local` игнорируются git
- Проверить `title` и `description` в `index.html`
- Проверить корректность шрифтов `Tektur` и `Roboto`
- Проверить production-сборку `npm run build`
- Проверить автотесты `npm run test`
