# Admin Operations

## SDD-6. Workflow заявок в admin

### Цель

Менеджер сопровождает заявку до завершения.

### Основной сценарий

1. Менеджер открывает `/admin`.
2. Ищет заявку по имени, контакту, формату, `objectSlug` или комментарию.
3. Фильтрует список по статусу.
4. Меняет статус заявки.
5. Добавляет комментарий менеджера.
6. Сохраняет workflow.

### Статусы

- `new`
- `contacted`
- `in_progress`
- `confirmed`
- `done`
- `canceled`

### Ожидаемый результат

- статус и комментарий сохраняются в SQLite
- `updatedAt` обновляется
- при необходимости сохраняется `lastContactAt`

### Артефакты реализации

- `GET /api/bookings`
- `PATCH /api/bookings/:id/workflow`
- `src/admin/AdminPage.tsx`

## SDD-7. Экспорт заявок в CSV

### Цель

Менеджер выгружает текущую выборку заявок для внешней обработки.

### Основной сценарий

1. Менеджер применяет поиск и фильтр по статусу.
2. Нажимает `Экспорт CSV`.
3. Frontend вызывает backend с текущими query-параметрами.
4. Backend возвращает CSV-файл.

### Ожидаемый результат

- экспорт содержит только текущую выборку
- в CSV есть ключевые поля заявки
- файл скачивается из admin-панели

### Артефакты реализации

- `GET /api/bookings/export.csv`
- `src/admin/AdminPage.tsx`

## SDD-8. CRUD объектов в admin

### Цель

Менеджер поддерживает каталог объектов в актуальном состоянии.

### Основной сценарий

1. Менеджер открывает список объектов в `/admin`.
2. Выбирает существующий объект или создаёт новый.
3. Заполняет поля карточки объекта.
4. Сохраняет изменения.
5. При необходимости удаляет объект.

### Ожидаемый результат

- объект доступен на лендинге и на карте
- каталог и admin работают из единого backend-источника

### Данные объекта

- `slug`
- `name`
- `city`
- `industry`
- `audience`
- `price`
- `duration`
- `blurb`
- `tags`
- `coordinates`
- `address`
- `safety`
- `format`
- `schedule`
- `audienceDetails`
- `includes`
- `highlights`
- `fullDescription`

### Артефакты реализации

- `GET /api/objects`
- `POST /api/objects`
- `PUT /api/objects/:slug`
- `DELETE /api/objects/:slug`
