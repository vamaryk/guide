# Admin Auth

## SDD-5. Вход администратора

### Цель

Менеджер получает доступ к защищённой admin-панели.

### Предусловия

- настроены `ADMIN_LOGIN` и `ADMIN_PASSWORD_HASH`

### Основной сценарий

1. Менеджер открывает `/admin`.
2. Вводит логин и пароль.
3. Система проверяет пароль по hash.
4. Backend создаёт admin-сессию.
5. Frontend получает token и открывает панель управления.

### Негативный сценарий

1. Логин или пароль неверны.
2. Система возвращает `401`.
3. Пользователь остаётся на форме входа.

### Ожидаемый результат

- доступ к admin-операциям без токена невозможен
- plain-text пароль не хранится в коде

### Артефакты реализации

- `server/auth.ts`
- `server/hash-password.ts`
- `server/index.ts`
- `src/admin/AdminPage.tsx`
