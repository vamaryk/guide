# Booking

## SDD-4. Бронирование объекта

### Цель

Гость отправляет рабочую заявку на посещение объекта.

### Основной сценарий

1. Гость открывает форму бронирования.
2. Заполняет имя, контакт, формат, дату и объект.
3. Подтверждает согласие.
4. Отправляет форму.
5. Система валидирует данные и сохраняет заявку.

### Ожидаемый результат

- создаётся запись со статусом `new`
- менеджер может увидеть заявку в `/admin`
- гость получает сообщение об успешной отправке

### Данные заявки

- `id`
- `name`
- `contact`
- `format`
- `date`
- `objectSlug`
- `consent`
- `status`
- `managerComment`
- `lastContactAt`
- `updatedAt`
- `createdAt`

### Артефакты реализации

- `POST /api/bookings`
- `server/db.ts`
- `server/types.ts`
