# SDD Overview

## Что здесь считается SDD

Для этого проекта `SDD` используется как `Scenario-Driven Development`: поведение системы описывается через пользовательские и операционные сценарии, а требования, API, данные и проверка привязываются к этим сценариям.

## Контекст продукта

Продукт помогает гостю:
- найти промышленный объект Саратовской области
- изучить краткую и подробную информацию по объекту
- перейти от карты к каталогу и бронированию
- отправить заявку на посещение

Продукт помогает менеджеру:
- авторизоваться в `/admin`
- просматривать заявки
- вести workflow заявки
- экспортировать текущую выборку в CSV
- управлять каталогом объектов

## Акторы

- `Гость`: посетитель лендинга
- `Менеджер`: администратор, работающий через `/admin`
- `Система`: frontend, backend API, SQLite

## Нефункциональные требования

- мобильная ширина `390px`, `375px`, `360px`, `320px` без горизонтального скролла
- фиксированное меню доступно при любом скролле
- карта должна иметь fallback при отсутствии API-ключа
- все admin-операции защищены токеном
- состояние каталога и заявок хранится в SQLite

## Definition of Done по SDD

Сценарий считается завершённым, если:
- пользовательский поток реализован end-to-end
- есть понятный позитивный результат для пользователя
- данные сохраняются в backend там, где это требуется
- для сценария есть хотя бы одна автоматическая проверка или явный чек в `docs/site-checklist.md`
